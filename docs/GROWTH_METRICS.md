# Growth Metrics

Минимальный набор метрик для роста проекта:

## Продуктовые метрики

- `PyPI downloads / month`
- `PyPI downloads by version`
- `Install -> first successful API call` (внутренняя метрика)

## Open-source метрики

- GitHub stars
- Forks
- Issues opened / closed
- Median time to first response in issues
- PR merge lead time

## Релизные метрики

- Release frequency (релизы в месяц)
- CI pass rate
- PyPI publish success rate

## Рекомендованный cadence

- Еженедельно:
  - обновлять top-3 проблемы из issues,
  - проверять download-тренд PyPI.
- Ежемесячно:
  - делать changelog summary,
  - публиковать 1 практический пример/кейc.
